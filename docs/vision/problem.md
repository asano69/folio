# Problem

- 現在のオープンソース（OSS）書籍ビューアの多くは、基本的な表示・整理機能に特化しており、研究・探究活動に不可欠なメモ、マーカー、タグ付けといった注釈機能の統合が不十分である。
- 技術的側面では、シングルページアプリケーション（SPA）として構築されたシステムが主流であり、各ページに対するURIの永続性や参照可能性への配慮が欠けている。このため、特定の箇所へのリンクが将来的に損なわれるリスクを内包しており、研究・探究活動の情報基盤としての信頼性に課題が残る。
- こうした開発状況の背景には、欧米諸国において書籍のデジタル化に対する法的な制限が厳しく、日本のような独自の自炊文化や、デジタル資料をOSSで積極的に利活用しようとする文化が醸成されにくいという社会的要因も推察される。
- したがってデジタル書籍資料を単なる消費財ではなく、永続的な知的資源として扱うための新たな情報システムの開発が必要とされている。
